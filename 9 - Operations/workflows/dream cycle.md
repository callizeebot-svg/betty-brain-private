# Dream Cycle

## Purpose

Nightly maintenance workflow for Betty Brain.

The dream cycle is how Betty keeps memory strong instead of merely accumulating files.

It should preserve the strongest Evolving Brain idea:

- compile
- connect
- reconcile
- strengthen

rather than just file things away.

## Goals

On a recurring cadence, the dream cycle should:

- review recent live logs and newly added notes
- identify repeated themes and patterns
- surface contradictions or stale assumptions
- strengthen links between projects, areas, people, identity, and operations
- suggest promotions from raw signal into durable pages
- identify duplication or fragmentation that should be cleaned up

## Inputs

Typical inputs include:

- recent entries in `2 - Live Logs/`
- new or changed project notes
- new or changed area notes
- new research summaries
- operations run notes
- onboarding or migration outputs

## Outputs

The dream cycle should produce one or more of:

- updates to project or area state notes
- suggestions for identity/aspirations updates
- contradiction notes for human review
- cleanup suggestions for duplicated docs
- a short run note in `9 - Operations/runs/`
- optional append to a review or reflections log later

## Rules

- do not silently rewrite historical events
- do not auto-edit identity claims without strong evidence and human review
- preserve attribution and evidence trails
- prefer small reliable maintenance over grand speculative rewriting
- do not move secrets or local auth state into shared memory

## Recommended cadence

- nightly or several times per week once the shared brain is active enough

## Suggested steps

1. Review the last meaningful changes in `2 - Live Logs/`
2. Check whether they imply updates to projects, areas, people, or operations docs
3. Identify repeated topics or unresolved tensions
4. Flag contradictions between current-state pages and recent evidence
5. Suggest or apply small safe updates to durable docs
6. Record the run and any unresolved issues

## Relationship to other workflows

- `windows-health-security-review.md` keeps the runtime healthy
- `dream-cycle.md` keeps the memory healthy
- future briefing/review workflows can read the results
