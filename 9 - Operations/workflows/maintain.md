# Maintain

## Purpose

Perform a periodic health check on Betty Brain itself.

This is about memory and system hygiene, not just runtime security.

## Goals

- identify stale docs
- identify duplicated or fragmented knowledge
- check whether workflows still make sense
- keep the brain coherent and navigable
- reduce drift between reality and current-state pages

## Inputs

- project docs
- area docs
- live logs
- workflow docs
- skill registry
- connector inventory
- operations notes

## Outputs

- cleanup suggestions
- small safe documentation fixes
- identified contradictions
- recommendations for structural improvements

## Steps

1. Review recently touched durable docs
2. Look for overlap, duplication, and stale assumptions
3. Check whether key registries and inventories are still current
4. Compare current-state docs with recent evidence
5. Make small safe fixes or note larger issues for review

## Rules

- prefer small reliable cleanup over large speculative restructuring
- do not erase useful historical context
- preserve source-of-truth boundaries
- flag larger architecture tensions instead of hiding them

## Recommended cadence

- weekly or fortnightly
