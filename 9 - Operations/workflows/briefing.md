# Briefing

## Purpose

Produce a concise briefing that helps Cally quickly understand what matters now.

## Goals

A good briefing should surface:

- active priorities
- recent important changes
- upcoming decisions or risks
- open loops worth attention
- anything Betty has been doing on Cally's behalf

## Inputs

Typical sources:

- recent `2 - Live Logs/` entries
- project current-state docs
- operations run notes
- recent assistant actions
- upcoming scheduled or time-sensitive items

## Output

A concise digest that answers:

- what changed?
- what matters now?
- what should happen next?

## Steps

1. Review recent changes in live logs and operations notes
2. Identify the highest-signal developments
3. Filter out noise and repetition
4. Group by priority or workstream
5. Present a short digest with recommended focus

## Rules

- do not turn the briefing into a wall of notes
- prefer signal over completeness
- highlight blockers, risks, and decisions clearly
- include Betty's own recent meaningful actions when useful

## Recommended cadence

- mornings once the shared brain becomes active enough
- on demand when a status reset is useful
