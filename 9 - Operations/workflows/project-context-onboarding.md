# Project Context Onboarding

## Purpose

Absorb a new project's context into Betty Brain without losing important signal or prematurely over-structuring it.

This will be especially important for IMatch and future incoming projects.

## Goals

- understand the incoming project materials
- extract durable project knowledge
- create or update the right project pages
- connect the project to skills, operations, and memory

## Inputs

- project overview
- specs
- code/assets
- user notes
- related research or history

## Outputs

- project README/current-state/vision or equivalent docs
- links to relevant skills and workflows
- live-log entries for major onboarding events
- clear next actions

## Steps

1. inspect incoming materials
2. identify the project's purpose, status, and constraints
3. create or update the project dossier docs
4. connect the project to relevant shared skills/workflows
5. note open questions and next decisions

## Rules

- do not assume a project is well-defined just because many files were provided
- preserve raw context until synthesis is complete
- avoid mixing project-specific runtime details into global memory docs unless broadly relevant

## Recommended cadence

- on demand when a new project or major project dump arrives
