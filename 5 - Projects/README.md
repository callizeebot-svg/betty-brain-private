# Projects

This folder holds durable project dossiers.

## What belongs here

Use project notes for:

- current project state
- goals and constraints
- risks and tensions
- next actions
- timeline/evidence of meaningful changes

## Rule of thumb

If a note describes the current best understanding of a project, it belongs here.
If it is still raw, route it through `.inbox/` first.

## Storage rule

Project understanding belongs here.

The actual shipped product/codebase often belongs in its own separate local repo and GitHub repo, with this folder holding the durable dossier, decisions, references, and links.
