# How to Use Betty Brain - Simple

This is the plain-English version.

## What Betty Brain is

Betty Brain is a shared brain stored in this repo.

Think of it like this:

- the **repo** is the memory
- **OpenClaw** keeps the system alive and coordinated
- **Claude Code** helps with deep technical work
- **Obsidian** lets you read and shape the brain visually

## How Betty should work

Good flow:

`capture -> route -> process -> log -> strengthen`

That means:

1. something new appears
2. it gets captured
3. Betty routes it to the right place
4. important parts become durable notes
5. review workflows make the brain stronger over time

## Why this is better than normal chat memory

Normal chat is easy to lose.

Betty Brain makes useful work durable:

- decisions become notes
- research becomes reusable
- workflows become repeatable
- memory survives tool changes

## How to keep costs sane

Do not make one giant session do everything forever.

Better pattern:

- keep Betty's main session strategic
- load only the files needed for the task
- use focused helper sessions for deep narrow work
- write the useful outcome back into the repo

See [[9 - Operations/workflows/context-budgeting-and-session-strategy]].

## Versioning rule

Important Betty changes should be saved in git.

That means:

- update the files
- commit important setup or memory changes
- push them to the private repo when appropriate

This gives Betty an inspectable history of how the brain changed.

See [[9 - Operations/workflows/versioning-and-sync]].

## Short version

Betty Brain works when:

- raw things are captured
- important things are processed
- durable truth lives in markdown
- Betty stays lean and delegates smartly
- important changes get versioned

That is how Betty stops being just a chat assistant and starts becoming a real working brain.
