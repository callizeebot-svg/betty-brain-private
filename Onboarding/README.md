# Onboarding

This folder helps Betty move from empty or partial setup to a working shared brain.

## Current role in Betty Brain

Right now, onboarding is partly complete:

- core identity exists
- core operations scaffolding exists
- skill and runtime architecture is being established
- shared memory architecture is now being formalized

## What onboarding should still cover

- absorbing more personal/project context over time
- importing useful existing notes or docs into `.inbox/`
- processing those materials into live logs and durable pages
- establishing recurring workflows and maintenance loops

## Key onboarding workflows

- `WINDOWS-INSTANCE-HANDOFF.md`
- `9 - Operations/workflows/inbox-processing.md`
- `9 - Operations/workflows/project-context-onboarding.md`
- `9 - Operations/workflows/shared-brain-migration.md`

## Rule

Onboarding should not end at first setup.
For Betty, onboarding is also the controlled intake of new important context as the brain grows.
