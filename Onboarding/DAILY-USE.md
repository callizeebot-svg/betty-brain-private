# Daily Use

This is the practical "how do I actually use Betty Brain?" note.

## Morning default

1. Open the Obsidian vault at `C:\Users\calli\Repos\betty-brain-private`
2. Open [[BRAIN MAP]]
3. Open [[9 - Operations/schedule]]
4. Check whether Betty surfaced anything from the morning briefing
5. If needed, open [[5 - Projects/Betty Brain]] to see current status and next actions

## During the day

### When you have a new thing

Ask:

- is this raw and unsorted?
- is this a dated event?
- is this stable project context?

If unsure, use [[9 - Operations/workflows/capture-routing]].

### Default moves

- rough thing -> `.inbox/`
- dated action/decision/learning -> `2 - Live Logs/`
- durable project context -> `5 - Projects/`
- operational rule/workflow -> `9 - Operations/`

## Weekly default

1. Review [[8 - North Star/NORTH STAR]]
2. Review recent live logs
3. Review [[5 - Projects/Betty Brain]] and any active project pages
4. Run or read [[9 - Operations/workflows/weekly-review]]

## When the system feels confusing

## When Betty feels expensive or overloaded

Check:

- [[9 - Operations/workflows/context-budgeting-and-session-strategy]]

Short rule:

- keep Betty main-session work strategic
- use fresh helpers for narrow deep work
- write durable outcomes back into the repo

## What Betty Brain is trying to become

Not just a note vault.
A working memory system where:

- raw captures become structured memory
- recurring workflows keep the brain alive
- human and agent work leave inspectable traces
- durable understanding survives tool changes
